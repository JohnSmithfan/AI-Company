# ReportEngine — Report Generator

> Agent 4/5: Report Generator | Documentation Specialist
> Engine: LLM Templates | Pipeline Stage4: Output

---

## Role

**ReportEngine** transforms structured insights from InsightEngine into polished, professional reports. It automatically selects the best report template based on analysis type and output format, then generates publication-ready documents in HTML or Markdown.

## Core Responsibilities

1. **Template Selection** — Automatic template matching based on:
   - Analysis depth (quick/standard/deep)
   - Industry domain (tech, finance, consumer, healthcare, etc.)
   - Crisis level (normal/attention/critical)
   - Output format (html/markdown/json)
   - Audience type (executive/analyst/public)

2. **HTML Report Generation** — Produces rich interactive reports:
   - Executive summary dashboard
   - Sentiment trend charts (time-series line charts)
   - Platform breakdown visualizations (bar/pie charts)
   - Word cloud of top keywords
   - Hot topic cards with sentiment badges
   - Risk alert panel (color-coded severity)
   - Detailed finding sections per platform
   - Methodology and data sourcing appendix

3. **Markdown Report Generation** — For integration and version control:
   - Structured headers following academic report format
   - Tables for quantitative data
   - Code blocks for raw data excerpts
   - Proper citation of sources

4. **File Count Baseline Management** — Tracks output artifacts:
   - Report file path and size
   - Embedded assets (images, charts)
   - Metadata file with trace ID

5. **AIGC Compliance Markers** — Embeds in every report:
   - Header banner: "Generated by AI Sentiment Analysis"
   - Timestamp and analysis ID
   - Methodology disclaimer
   - Confidence disclosure statement

## Pipeline Position

```
[InsightEngine] Insights → [ReportEngine] → Report File (HTML/MD)
                                ↓
                         [template selection]
                         [HTML generation]
                                ↓
                         [ForumEngine] (validation)
```

## Configuration

| Parameter | Environment Variable | Default | Description |
|-----------|---------------------|---------|-------------|
| Output Dir | `REPORT_OUTPUT_DIR` | ./reports | Report output directory |
| Template Dir | `REPORT_TEMPLATE_DIR` | ./templates | Custom template directory |
| LLM API Key | `REPORT_API_KEY` | (required) | Report generation LLM key |
| LLM Base URL | `REPORT_BASE_URL` | (provider) | LLM endpoint |
| LLM Model | `REPORT_MODEL_NAME` | qwen-plus | Report writing model |
| Chart Library | `CHART_RENDERER` | static-svg | Chart rendering engine |

## Error Codes

| Code | Message | Resolution |
|------|---------|------------|
| REPORT_001 | Template selection failed | Check available templates for analysis type |
| REPORT_002 | HTML generation error | Validate template structure integrity |
| REPORT_003 | Report file write permission denied | Check OUTPUT_DIR permissions |

## Integration Points

- **Upstream**: Receives structured insights from InsightEngine
- **Output**: Generates report files to configured output directory
- **Downstream**: Report passed to ForumEngine for collaborative discussion

## AIGC Requirements

- Every report MUST include visible AIGC disclosure header
- Footer: "This report was generated by AI (sentiment-analysis ReportEngine). Verify critical findings before action."
- Metadata block with: ai_generated, timestamp, model_version, trace_id

## Constraints

- All reports MUST include AIGC labeling
- PII masking before report generation
- All API credentials in environment variables
- No dynamic code execution in templates
- File write permissions verified before output
