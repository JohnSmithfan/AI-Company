# Workflow Execution — Sentiment Analysis Team

> Step-by-step execution procedures for the full sentiment analysis pipeline.

---

## Full Pipeline Workflow

### Phase 1: Initialization

```
Step 1.1: Validate user input
  - Check query is not empty (min 2 chars)
  - Validate platforms enum
  - Validate date_range (start <= end, max 365 days)
  - Validate analysis_depth (quick/standard/deep)
  - Validate output_format (html/markdown/json)

Step 1.2: Initialize agent contexts
  - Load QueryEngine configuration
  - Load MediaEngine configuration
  - Load InsightEngine configuration
  - Load ReportEngine configuration
  - Load ForumEngine configuration

Step 1.3: Generate trace_id
  - UUID-based unique identifier
  - Format: SA-[YYYYMMDD]-[8-char-hex]
```

### Phase 2: Data Collection (QueryEngine)

```
Step 2.1: Keyword optimization
  - Synonym expansion (Chinese + English)
  - Trending term injection
  - Platform-specific adaptation

Step 2.2: Execute search (parallel)
  - tavily_search (general)
  - tavily_news_search (news)
  - tavily_extract (URLs)
  - Rate limiting: 100/min

Step 2.3: Deduplicate results
  - URL hashing
  - Content similarity (>0.9 threshold)
  - Remove duplicates

Step 2.4: Normalize data
  - Standard schema (source, url, title, content, published_at, author)
  - Timestamp normalization to ISO8601
  - Relevance scoring

Output: raw_data array (>=5 items)
```

### Phase 3: Content Analysis (MediaEngine)

```
Step 3.1: Batch processing preparation
  - Split data into batches (max 50 per batch)
  - Priority queue for crisis detection

Step 3.2: Text analysis (parallel)
  - Sentiment classification (positive/negative/neutral)
  - Emotion detection
  - Intensity scoring

Step 3.3: Multimodal analysis
  - Image analysis (if applicable)
  - Video analysis (if applicable)
  - Logo/brand recognition

Step 3.4: Crisis detection
  - Keyword scanning
  - Severity classification (normal/attention/critical)
  - Alert generation

Output: annotated_content array
```

### Phase 4: Deep Mining (InsightEngine)

```
Step 4.1: Historical query (parallel)
  - Connect to MediaCrawlerDB
  - Query last 30 days data
  - Entity extraction

Step 4.2: Correlation analysis
  - Compare current vs historical baseline
  - Calculate delta
  - Trend direction identification

Step 4.3: Trend identification
  - Volume trends
  - Sentiment trends
  - Engagement trends
  - Geographic spread

Step 4.4: Entity analysis
  - Brand sentiment scores
  - Product mention tracking
  - Person/org extraction

Output: structured_insights object
```

### Phase 5: Report Generation (ReportEngine)

```
Step 5.1: Template selection
  - Match analysis_depth
  - Match crisis level
  - Match audience type
  - Match output_format

Step 5.2: Content population
  - Executive summary dashboard
  - Sentiment trend charts
  - Platform breakdown
  - Hot topics
  - Risk alerts
  - Detailed findings

Step 5.3: AIGC compliance
  - Header: "Generated by AI"
  - Metadata block: ai_generated, timestamp, trace_id
  - Footer disclaimer

Step 5.4: File output
  - Write to REPORT_OUTPUT_DIR
  - Create metadata file
  - Validate write success

Output: report_file path
```

### Phase 6: Collaborative Discussion (ForumEngine)

```
Step 6.1: Forum initialization
  - Load report from ReportEngine
  - Assign agent perspectives:
    - Bullish Analyst (positive)
    - Bearish Analyst (critical)
    - Neutral Analyst (balanced)
    - Contrarian (alternative)

Step 6.2: Discussion rounds
  - Round 1: Opening statements
  - Round 2: Evidence exchange
  - Round 3: Synthesis

Step 6.3: Divergence summary
  - Identify agreement points
  - Identify disagreement points
  - Moderator assessment
  - Recommended actions

Output: discussion_transcript
```

### Phase 7: Finalization

```
Step 7.1: Integrate discussion into report
  - Add divergence summary to report
  - Update AIGC metadata

Step 7.2: Write memory note
  - Daily: .workbuddy/memory/YYYY-MM-DD.md
  - Long-term: MEMORY.md
  - Include: trace_id, entities, sentiment scores, alerts

Step 7.3: Return results
  - report_file path
  - discussion_transcript
  - summary metrics
  - trace_id for audit
```

```
├── REPORT_003

    └── Check LLM endpoint → Retry → Skip forum
```
