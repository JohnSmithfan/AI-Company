## 02: Robustness Checks

### Input Validation Checklist

- [ ] Query not empty (min 2 characters)
- [ ] Platform enum valid (weibo, xiaohongshu, douyin, kuaishou, wechat, baidu, zhihu, toutiao, bilibili, news, all)
- [ ] Date range valid (start <= end, max 365 days)
- [ ] Analysis depth in (quick, standard, deep)
- [ ] Output format in (html, markdown, json)

### Security Validation Checklist

- [ ] No eval() / exec() in any template
- [ ] No unauthorized network calls (whitelist: Tavily, Bocha, LLM endpoints)
- [ ] No sensitive path access (~/.ssh, ~/.aws, /etc, C:/Windows)
- [ ] API credentials in environment variables only
- [ ] PII masking applied before any output

### AIGC Compliance Checklist

- [ ] Report header includes "Generated by AI"
- [ ] Metadata block with ai_generated: true
- [ ] Timestamp included in all outputs
- [ ] Trace ID generated for audit trail
- [ ] Footer disclaimer present

## 05: Workflow Execution

### Full Sentiment Analysis Pipeline

```
Step 1: QueryEngine (Data Collection)
  - Receive user query and parameters
  - Execute Tavily API search
  - Normalize and deduplicate results
  - Output: Raw data array

Step 2: MediaEngine (Content Analysis)
  - Receive raw data from QueryEngine
  - Execute multimodal analysis (text/image/video)
  - Detect crisis signals
  - Output: Annotated content

Step 3: InsightEngine (Deep Mining)
  - Receive annotated content from MediaEngine
  - Query historical database (parallel)
  - Perform trend identification
  - Output: Structured insights

Step 4: ReportEngine (Report Generation)
  - Receive structured insights from InsightEngine
  - Select appropriate template
  - Generate HTML/Markdown report
  - Embed AIGC disclosure
  - Output: Report file

Step 5: ForumEngine (Validation)
  - Receive report from ReportEngine
  - Initialize multi-agent discussion
  - Facilitate structured debate
  - Generate divergence summary
  - Output: Discussion transcript
```
